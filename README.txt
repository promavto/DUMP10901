README дампа1090
===

Dump 1090 — это декодер Mode S, специально разработанный для устройств RTLSDR.

Основные особенности:

* Надежное декодирование слабых сообщений, в режиме 1090 наблюдали многие пользователи.
   улучшенный диапазон по сравнению с другими популярными декодерами.
* Поддержка сети: поток TCP30003 (MSG5...), необработанные пакеты, HTTP.
* Встроенный HTTP-сервер, который отображает обнаруженные в данный момент самолеты.
   Google Map.
* Исправление однобитовых ошибок с использованием 24-битной CRC.
* Возможность декодирования сообщений DF11, DF17.
* Возможность декодирования форматов DF, таких как DF0, DF4, DF5, DF16, DF20 и DF21.
   где контрольная сумма сравнивается с адресом ИКАО путем перебора
   Поле контрольной суммы с использованием недавно просмотренных адресов ИКАО.
* Декодировать необработанные образцы IQ из файла (используя ключ командной строки --ifile).
* Интерактивный режим командной строки, в котором обнаружены самолеты в данный момент.
   отображаются в виде списка, обновляющегося по мере поступления новых данных.
* Расшифровка координат CPR и расчет трека по скорости.
* Потоковая передача TCP-сервера и получение необработанных данных к подключенным клиентам и от них.
   (используя --net).

Хотя время от времени я все еще добавляю/исправляю что-то в своем форке, я нацелен
минимализм реализации. Однако существует
[полная вилка с гораздо большим количеством функций] (https://github.com/MalcolmRobb/dump1090)
доступен, разработан MalcolmRobb.

Монтаж
---

Введите «сделать».

Обычное использование
---

Для захвата трафика непосредственно с вашего устройства RTL и отображения захваченного трафика
в стандартном выводе просто запустите программу вообще без опций:

     ./dump1090

Чтобы просто вывести шестнадцатеричные сообщения:

     ./dump1090 --raw

Чтобы запустить программу в интерактивном режиме:

     ./dump1090 --интерактивный

Запустить программу в интерактивном режиме с поддержкой сети и подключиться
в браузере перейдите по адресу http://localhost:8080, чтобы увидеть текущий трафик:

     ./dump1090 --interactive --net

В интерактивном режиме можно получить менее информативную, но более информативную информацию.
Вывод в «аркадном стиле», при котором экран обновляется каждую секунду, отображая
все недавно замеченные самолеты с некоторой дополнительной информацией, такой как
высота и номер рейса, извлеченные из полученных пакетов режима S.

Использование файлов в качестве источника данных
---

Чтобы декодировать данные из файла, используйте:

     ./dump1090 --ifile /путь/к/binfile

Бинарный файл должен быть создан с использованием `rtl_sdr`, вот так (или с любым другим
программа, способная выводить 8-битные беззнаковые выборки IQ с частотой дискретизации 2 МГц).

     rtl_sdr -f 1090000000 -s 2000000 -g 50 выходной.bin

В примере `rtl_sdr` используется коэффициент усиления 50, просто вам следует использовать самый высокий
усиление, доступное для вашего тюнера. Это не требуется при вызове самого Dump1090.
поскольку он может автоматически выбирать максимальное усиление, поддерживаемое.

Можно передать программе данные через стандартный ввод, используя
опция --ifile с аргументом «-».

Дополнительные опции
---

Dump1090 можно вызвать с другими параметрами командной строки, чтобы установить другой
усиление, частота и так далее. Для списка опций используйте:

     ./dump1090 --help

Все, что здесь не описано, должно быть очевидным, и для большинства пользователей, звонящих
лучше всего вообще без аргументов.

Надежность
---

По умолчанию Dump1090 пытается исправить однобитовые ошибки с помощью контрольной суммы.
По сути, программа попытается перевернуть каждый бит сообщения и проверить,
контрольная сумма полученного сообщения совпадает.

Это действительно способно исправлять ошибки и, по моему опыту, работает надежно.
однако, если вас интересуют очень надежные данные, я предлагаю использовать
ключ командной строки --no-fix, чтобы отключить исправление ошибок.

Характеристики и чувствительность обнаружения
---

По моему ограниченному опыту, Dump1090 смог декодировать большое количество сообщений.
даже в условиях, когда я сталкивался с проблемами при использовании других программ, однако
никаких официальных тестов не проводилось, поэтому я не могу утверждать, что эта программа
лучше или хуже по сравнению с другими подобными программами.

Если вы можете захватить трафик, который Dump1090 не может правильно декодировать, отбросьте
мне письмо со ссылкой для скачивания. Я могу попытаться улучшить обнаружение во время
мое свободное время (это всего лишь хобби-проект).

Возможности сетевого сервера
---

Включив поддержку сети с помощью --net, Dump1090 начинает прослушивать
для клиентских подключений через порт 30002 и 30001 (вы можете изменить оба
порты, если хотите, смотрите вывод --help).

Порт 30002
---

Подключенным клиентам данные предоставляются как можно скорее по мере их поступления с устройства.
(или из файла, если используется --ifile) в необработанном формате, аналогичном следующему:

     *8D451E8B99019699C00B0A81F36E;

Каждая запись отделяется простой новой строкой (символ LF, шестнадцатеричный код 0x0A).

Порт 30001
---

Порт 30001 является портом необработанного ввода и может использоваться для подачи Dump1090
данные в том же формате, что указан выше, с шестнадцатеричными сообщениями, начинающимися с
`*` и заканчивающийся символом `;`.

Например, если есть другой удаленный экземпляр Dump1090, собирающий данные.
можно суммировать вывод в локальный экземпляр Dump1090, делая что-то
так:

     nc удаленный-dump1090.example.net 30002 | NC локальный хост 30001
